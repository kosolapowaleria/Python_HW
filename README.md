# Документация проекта

## Описание базового синтаксиса и записи и форматирования

Проект использует следующие технологии и синтаксис:

1. **Python:** Основной язык программирования для написания тестов.
2. **Selenium:** Библиотека для автоматизации взаимодействия с веб-браузером.
3. **Pytest:** Фреймворк для написания и запуска тестов.
4. **Allure:** Инструментов для генерации отчётов о выполнении тестов.

### Форматирование кода

— Код форматируется в соответствии с PEP8 (стиль написания кода на Python).

— Используются docstrings для документирования методов и функций.

— Все шаги теста размечаются с помощью `allure.step` или `with allure.step` для улучшения читаемости отчёта.

___

## Автоматизированные UI‑тесты

Проект содержит два набора автоматизированных UI‑тестов для разных веб‑приложений:

### 1. Интернет‑магазин Saucedemo

Набор тестов для веб‑приложения [Saucedemo](https://www.saucedemo.com/)

**Описание проекта**

Тесты покрывают полный цикл покупки:
* авторизация в системе;
* добавление товаров в корзину;
* проверка содержимого корзины;
* оформление заказа;
* проверка итоговой суммы.

### 2. Веб‑калькулятор (Slow Calculator)

Набор тестов для веб‑калькулятора с задержкой вычислений [Slow Calculator](https://bonigarcia.dev/selenium-webdriver-java/slow-calculator.html)

**Описание проекта**

Тесты проверяют корректность математических операций в калькуляторе с настраиваемой задержкой отображения результата:
* установка времени задержки вычислений;
* выполнение базовых арифметических операций (сложение, умножение и т. д.);
* проверка результата вычислений;
* работа с очисткой дисплея.

## Инструкция по запуску текстов для формирования отчёта в Allure

1. Установите необходимые зависимости:
```bash
pip install -r requirements.txt
```

2. Запустите тесты с генерацией отчёта Allure:
```bash 
pytest --alluredir=./allure-results
```

Эта команда запустит все тесты и сохранит результаты в директорию `./allure-results`

3. Опция запуска тестов:
* запуск только тестов калькулятора:
```bash
pytest pytest 10_lesson/test_calculator_PO.py --alluredir=./allure-results
```
* запуск только тестов онлайн-магазина:
```bash
pytest 10_lesson/test_online_shop_PO.py --alluredir=./allure-results
```

## Инструкция по просмотру сформированного отчёта в Allure

1. После завершения тестов, чтобы посмотреть отчёт, выполните команду:
```bash
allure serve ./allure-results
```

Эта команда запустит локальный сервер и откроет отчёт в браузере.

2. В отчёте вы увидите: 

 **Название тестов:**
* Для Saucedemo: Полный сценарий покупки в онлайн-магазине.
* Для калькулятора: Тестирование веб‑калькулятора: сложение и т.д. 

 **Описание тестов:** 
* Для Saucedemo: Описание, что текст проверяет полный цикл покупки:
авторизация → корзина → заказ → сумма.
* Для калькулятора: Калькулятор: проверка математической операции с заданной задержкой и т.д.

**Шаги тестов:**

Для Saucedemo:
* Авторизация в система
* Добавление товаров в корзину
* Переход в корзину
* Проверка наличия товаров в корзине
* Переход к оформлению заказа
* Заполнение персональных данных
* Подтверждение заказа
* Получение и проверка итоговой суммы
* Проверка корректности суммы заказа

Для калькулятора:
* Установка задержки вычислений.
* Ввод математического выражения (например, 7+8 или 5*3).
* Выполнение расчёта (нажатие кнопки =).
* Ожидание результата на дисплее.
* Получение и проверка результата.
* Очистка дисплея (для соответствующего теста).

— **Результат:**

По завершении выполнения тестов отчёт Allure содержит детализированную информацию о результатах для каждого шага и тестов в целом.

Отчёт отображает один из двух сценариев:

**1. Успешное выполнение (`passed`):**
* все шаги теста отмечены как `passed`
* итоговая сумма заказа соответствует ожидаемой (для Saucedemo)
* результат вычисления соответствует ожидаемому (для калькулятора)
* в отчёте видна последовательность действий от начала до конца

**2. Ошибка (`failed`):**
* общий статус теста — `failed`
* шаг с ошибкой выделен красным, содержит сообщение из `assert`
* последующие шаги помечены как `skipped` с пояснением «Пропущен из‑за ошибки на предыдущем шаге»
* доступна трассировка ошибки (файл и строка кода).

**Важно:** статус `skipped` для **всех тестов** не возникает в текущей реализации. Он может появиться для **отдельных шагов**, если предыдущий шаг завершился ошибкой.
